# CLAUDE.md — Roatán Insider (iOS App)

## Project Overview

**Roatán Insider** is a native iOS app — a premium travel guide for Roatán, Honduras. It helps cruise passengers, vacationers, long-term travelers, and expats discover the island's best businesses, beaches, activities, and services through a curated directory, interactive map, travel tools, and insider guides.

**Owner:** Josh Green (Josh Green Design Studio), based in West Bay, Roatán
**App Name:** Roatán Insider
**Subtitle:** Your Local Guide to Roatán
**Tagline:** Explore the island like a local.

## Design System

### CRITICAL — READ THIS FIRST

This app must look like it was designed by a world-class human designer, NOT generated by AI. Study Apple.com, Nike.com, and Airbnb's iOS app for reference. The design language is:

- **Clean, minimal, confident.** When in doubt, remove something.
- **Photo-driven.** Large, beautiful photography dominates. UI chrome is minimal.
- **Generous whitespace.** Sections breathe. Nothing feels cramped.
- **Flat design.** No gradients, no drop shadows on cards, no beveled edges.
- **Strategic contrast.** Alternating white and dark charcoal sections like Apple.com.
- **Restrained color.** Pink accent used sparingly for CTAs and key actions only. Not everywhere.
- **Typography-forward.** Big, bold headlines with tight letter-spacing.

### Color Palette

```
PRIMARY ACCENT (Pink):     #E31B4E  — CTAs, primary buttons, active tab, key highlights ONLY
SECONDARY ACCENT (Mint):   #4ECDC4  — Stats, badges, secondary highlights, trust indicators
DARK (Charcoal):           #2D2D2D  — Dark section backgrounds, headlines on light backgrounds
NEAR BLACK:                #1A1A1A  — Tab bar background, deepest dark sections
WHITE:                     #FFFFFF  — Primary background, text on dark sections
OFF WHITE:                 #F8F8F8  — Subtle section differentiation, card backgrounds
LIGHT GRAY:                #999999  — Secondary body text, captions, metadata
MEDIUM GRAY:               #666666  — Body text on light backgrounds
GOLD STAR:                 #F5A623  — Ratings stars only
```

### Color Usage Rules

- Pink (#E31B4E) is used ONLY for: primary CTA buttons, the active tab bar icon, heart/favorite icons when active, and the app icon background. NOTHING ELSE. If you're tempted to use pink on something, don't.
- Mint (#4ECDC4) is used for: stat numbers, verified badges, secondary action buttons (outline style), and trust indicators. Also used sparingly.
- Dark charcoal sections alternate with white sections down the page, like Apple.com product pages.
- NO colored category badges. Categories are differentiated by icon, not color.
- NO rainbow of colors. The entire app should feel like 90% black/white/gray with occasional pink and mint accents.
- Map pins should ALL be the same color (pink #E31B4E), not a different color per category.

### Typography

- **Display/Headlines:** SF Pro Display, weight 700-800, large sizes (28-36pt), tight letter-spacing (-0.5 to -1.0)
- **Body:** SF Pro Text, weight 400, 16pt, comfortable line-height (1.5)
- **Captions/Metadata:** SF Pro Text, weight 400, 13-14pt, medium gray color
- **Button Labels:** SF Pro Text, weight 600, 16pt
- **Tab Labels:** SF Pro Text, weight 500, 10pt

### App Icon

- Background: Pink (#E31B4E)
- Foreground: Dark charcoal (#2D2D2D) palm tree silhouette
- The palm tree logo asset will be provided as an image in the project Assets catalog
- Rounded corners per iOS standard

### Component Patterns

**Cards (Business Listings):**
- NO borders, NO shadows, NO colored bars
- Large photo (aspect ratio 16:9 or 4:3) as the dominant element — takes up 60%+ of the card
- Rounded corners: 16pt
- Below photo: Business name (bold, dark), one line of metadata (area · price range), nothing else
- On the photo: subtle dark gradient at bottom for readability if overlaying text
- Favorite heart icon: top right of photo, white when unsaved, pink when saved
- Tap entire card to open detail page

**Buttons:**
- Primary: Pink (#E31B4E) background, white text, 50pt height, 16pt corner radius, full width
- Secondary: Transparent background, 1pt mint (#4ECDC4) border, mint text
- Ghost: No background, no border, pink or mint text only
- Minimum tap target: 44x44pt on everything

**Section Layout (Home page pattern):**
- White section → Dark charcoal section → White section → Dark section
- Each section has generous vertical padding (40-60pt)
- Headlines are large and bold, left-aligned
- Content follows below with comfortable spacing

**Navigation:**
- 5-tab TabView at the bottom
- Tab bar: near-black background (#1A1A1A), always visible
- Inactive icons: medium gray (#666666)
- Active icon: pink (#E31B4E)
- Icons: SF Symbols, weight .medium
- Labels below icons, always visible (not hidden on scroll)

**Detail Pages:**
- Full-width hero image at top (takes up ~40% of screen height)
- Content scrolls underneath the image
- No colored bars or badges — status shown as subtle text
- Contact actions (Call, WhatsApp, Directions) as a horizontal row of icon buttons, not big colored rectangles
- Insider Tip shown with a subtle left border accent in mint, not a full background box

## Business Data Schema

All data is bundled as JSON in the app bundle. No network calls needed.

```swift
struct Business: Identifiable, Codable, Hashable {
    let id: String                      // UUID string
    let slug: String                    // URL-friendly: "sundowners-bar"
    let name: String                    // "Sundowners Bar"
    let description: String             // 100-200 words
    let insiderTip: String?             // "Grab a dock seat by 4:30pm..."
    let category: Category              // .eat, .drink, .dive, etc.
    let subcategory: String             // "Beach Bars"
    let area: Area                      // .westBay, .westEnd, etc.
    let latitude: Double                // 16.XXXXXX
    let longitude: Double               // -86.XXXXXX
    let addressDescription: String      // "At the end of the main road in West End"
    let phone: String?
    let whatsapp: String?
    let email: String?
    let website: String?
    let facebook: String?
    let instagram: String?
    let priceRange: Int                 // 1-4 ($ to $$$$)
    let hours: [String: DayHours]       // ["monday": DayHours(open: "07:00", close: "17:00")]
    let features: [String]              // ["PADI Certified", "Beachfront", "Live Music"]
    let images: [String]                // Asset catalog image names
    let isVerified: Bool
    let isFeatured: Bool
    let status: String                  // "active", "paused", "closed"
}

enum Category: String, Codable, CaseIterable {
    case eat, drink, dive, tours, shop, stay, rentals, transport, beaches, nightlife
}

enum Area: String, Codable, CaseIterable {
    case westBay = "west_bay"
    case westEnd = "west_end"
    case sandyBay = "sandy_bay"
    case coxenHole = "coxen_hole"
    case flowersBay = "flowers_bay"
    case frenchHarbour = "french_harbour"
    case oakRidge = "oak_ridge"
    case puntaGorda = "punta_gorda"
    case portRoyal = "port_royal"
    case campBay = "camp_bay"
}
```

## Search & Filtering

Users can discover businesses through:

- **Text search:** Search across name, description, features, area. Use Swift's built-in search capabilities.
- **Category filter:** Filter by one or multiple categories
- **Area filter:** Filter by geographic area
- **Price range filter:** Filter by $ to $$$$
- **Feature filter:** e.g., "Family Friendly", "PADI Certified", "Live Music", "Beachfront"
- **Open Now:** Toggle based on current time vs. business hours
- **Sort:** Distance (requires GPS), Name A-Z, Featured first

## Authentication (Optional)

Auth is optional. The app works fully without signing in. Favorites save locally via SwiftData.

When a user signs in, favorites sync to the cloud (future backend integration). Auth options:

- Sign in with Apple (required for App Store if any auth is offered)
- Sign in with Google
- Email + Password

Auth is presented as a gentle nudge, not a gate. Example: "Sign in to sync your favorites across devices" — shown on the Saved tab if the user has favorites but no account.

## Content Voice

All descriptions, tips, and guide content should sound like a knowledgeable local friend:

- **Insider, not tourist:** "Locals call this the best baleada on the island"
- **Specific, not vague:** "Ask for Maria's coconut shrimp — it's not on the menu"
- **Honest, not promotional:** "The food is excellent but service can be slow on cruise ship days"
- **Practical, not flowery:** "Bring cash — no card reader"

## Important Roatán Context

- Roatán is a 48-mile-long island, 5 miles wide. West Bay and West End are the main tourist zones on the western tip.
- Two cruise ports: Mahogany Bay (near West Bay) and Town Center (Coxen Hole). They serve different areas.
- No traditional street addresses. Use GPS coordinates + landmark descriptions for every listing.
- USD widely accepted. Local currency is Honduran Lempira (HNL). ~1 USD = 24.85 HNL.
- Internet connectivity is unreliable outside tourist zones. Bundled data is essential.
- Cruise passengers have 6-8 hours on the island and zero patience. Fast, clear, decisive UI.

## Coding Standards

- Use SwiftUI exclusively. No UIKit unless absolutely necessary.
- MVVM architecture. Views observe ViewModels via @Observable (iOS 17).
- One view per file. File name matches the struct name.
- Use Swift's native types. No Objective-C bridging.
- All strings that appear in the UI must be in English.
- Use SF Symbols for all icons. No custom icon assets except the logo.
- Use @Environment, @State, @Binding appropriately. No unnecessary state.
- Keep views under 150 lines. Extract subviews into separate files.
- Use async/await for any asynchronous operations.
- Comment sparingly — code should be self-documenting. Only comment "why", not "what."

## Rules for Claude Code

- NEVER use gradients anywhere in the UI.
- NEVER use colored borders on cards.
- NEVER use drop shadows on cards (subtle shadows on floating elements like the tab bar are OK).
- NEVER make the UI look like a Bootstrap template or generic startup page.
- NEVER use more than 2 accent colors (pink + mint). Everything else is black, white, or gray.
- NEVER add features not described in this document without asking.
- NEVER refactor existing code unless explicitly asked.
- Always use Plan Mode for changes that touch more than 3 files.
- When creating seed data, use realistic Roatán businesses with accurate GPS coordinates.
- When in doubt about Roatán-specific details, ASK the user.
- The design must pass the "would a human designer approve this?" test on every screen.
